Demo Environment

Demo Setup

🖥️ Admin Machine (HQ side)

Standard laptop or desktop. Browser open to the TactiTok Admin Portal. Used for steps 1–2 (content upload). Content is immediately visible on the catalog.

📱 Edge Tablet (fighter side)

~10″ tablet running Chrome. Connected to the lab network. Shows the TactiTok consumer app. Used for steps 3–15. Screen is projected for all evaluators to see.

📶 Network Conditions

Local lab network simulating constrained bandwidth. Step 11 simulates a full disconnect by toggling tablet WiFi off. This is intentional — not a malfunction.

📦 Pre-loaded Content

15 demo items already in the catalog: 10 PDFs + 5 videos. Steps 1–2 add 2 additional items live. All content is representative training material.

Phase 1 — Admin Upload
01

HQ staff uploads a new video via the admin portal

What you'll see

  • Admin portal opens to the upload form
  • Staff fills in title, description, category, and interest tags
  • File picker accepts the MP4; content type is auto-detected
  • After submit: video appears in the catalog with metadata and thumbnail
Phase 1 — Admin Upload
02

HQ staff uploads a PDF and assigns categories & interests

What you'll see

  • Same upload form — now selecting a PDF file
  • Category selector shows the 2-level category tree
  • Interest tags are assigned (different from step 1)
  • After submit: PDF appears in catalog, organized under the chosen category
Phase 2 — Edge Consumer
03

Edge device opens the app for the first time

What you'll see

  • App loads fully in Chrome — no splash longer than 2 seconds
  • Interest selection screen appears before any content feed
  • Available interests shown as selectable chips or toggle buttons
Phase 2 — Edge Consumer
04

User selects 2–3 interests

What you'll see

  • Tapping a chip highlights it — selections are visually clear
  • Tapping "Continue" navigates to the Reels feed
  • Feed immediately filtered — only videos matching selected interests appear
Phase 2 — Reels
05

User scrolls through reels

What you'll see

  • First video auto-plays — no tap required — within 2 seconds
  • Swipe up: current video stops; next video auto-plays
  • Swipe down: returns to previous video
  • Overlay shows: title, description (truncated), Like, Save, Download buttons
Phase 2 — Reels
06

User taps "Download" on a video

What you'll see

  • Download progress indicator appears on the video overlay
  • On completion: button state updates (downloaded state)
  • Item is now cached on the edge proxy — ready for offline playback
Phase 2 — Library
07

User navigates to Library tab

What you'll see

  • Library view loads with the 2-level category tree
  • Tapping a category expands it and shows filtered content items
  • Video and PDF items are visually distinguishable (type icon/badge)
  • Search bar visible at the top of the library
Phase 2 — Library
08

User searches for a topic

What you'll see

  • Typing into the search bar filters results in real-time (or on submit)
  • Searches across title and description fields
  • Relevant items appear within seconds
  • Each result shows type badge (video / PDF) for quick identification
Phase 2 — Library
09

User opens a PDF from the library

What you'll see

  • PDF opens in the in-browser viewer — no external apps, no pop-ups
  • Page-by-page navigation works via tap or scroll
  • Entire experience stays inside Chrome
Phase 2 — Library
10

User downloads the PDF

What you'll see

  • Download button on the PDF viewer triggers a progress indicator
  • On completion: PDF saved locally (edge proxy cache)
  • PDF appears in the Downloads tab alongside the previously downloaded video
Phase 3 — Offline
11

Network disconnected (simulated)

What you'll see

  • Tablet WiFi turned off — network drop is intentional
  • Offline indicator appears in the app (banner or status badge)
  • Reels and Library tabs show graceful offline state — no crash
  • App remains stable and responsive
Phase 3 — Offline
12

User opens Downloads tab

What you'll see

  • Downloads tab loads instantly — no network needed
  • Both downloaded items listed: video (step 6) + PDF (step 10)
  • Each item shows: title, type (video / PDF), file size, download date
Phase 3 — Offline
13

User plays downloaded video offline

What you'll see

  • Video player opens from the Downloads tab
  • Plays immediately — no buffering, no network required
  • Full video playback from edge proxy cache — completely offline
Phase 3 — Offline
14

User opens downloaded PDF offline

What you'll see

  • PDF viewer opens from the Downloads tab
  • Renders fully — all pages available, no network required
  • Page navigation works exactly as when online
  • Everything stays inside Chrome — no external app
Phase 3 — Reconnect & Sync
15

Network restored; HQ uploads an updated video

What you'll see

  • WiFi turned back on — offline indicator disappears
  • App syncs the catalog automatically on reconnect
  • Downloaded item shows an "updated" badge — catalog version exceeds local version
  • Reels and Library return to normal online operation
