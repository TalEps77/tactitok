הקשר

מדוע האבטחה חשובה כאן

הבעיה

  • תוכן רגיש — חומר אימון מבצעי שאינו לגישה ציבורית
  • ללא זהות משתמש — קיוסק שדה הוא אנונימי מעיצוב
  • מכשיר שדה — אינו מנוהל על-ידי צוות הפיתוח
  • ללא DRM — האבטחה מבוססת על בידוד רשת + TLS

היקף אבטחת MVP

  • TLS על כל תעבורת ענן — תוכן לא עובר בטקסט ברור
  • אימות מנהל — רק צוות מורשה יכול לפרסם תוכן
  • ללא טלמטריה — לא יוצאים נתוני שימוש מהמכשיר
  • אימות תוכן — לא מתקבלים קבצים ניתנים להרצה
מחוץ להיקף MVP: נעילת קיוסק ברמת מערכת הפעלה, כללי חומת אש VM, אבטחה פיזית של המכשיר, DRM/סיווג תוכן — אלו אחריות תשתית.

ארכיטקטורת אמון

שלושה אזורי אמון

אזור 1

שרת — מהימן

VM ענן: Content API, Admin API, PostgreSQL, מאגר קבצים. מקור אמת סמכותי. כל הנתונים כאן. HTTPS מסתיים כאן. גישת הרשת מוגבלת.

אזור 2

מכשיר קצה — חצי-מהימן

קיוסק שדה: ללא זהות משתמש, גישה אנונימית, קטלוג קריאה בלבד. Chrome מתחבר לפרוקסי הקצה דרך localhost בלבד. מטמון מקומי מחזיק תוכן שנגשו אליו קודם. לא ניתן לשנות נתוני שרת.

אזור 3

לקוח מנהל — מאומת

מחשב מטה: סיסמה משותפת + טוקן סשן, Admin API בלבד, פועל ברשת המטה. הסשן נמחק בסגירת הלשונית (sessionStorage, לא localStorage).

הפניה לתרשים: DG-16 (אזורי אמון/אבטחה) מספק ייצוג ויזואלי של גבולות אלו.

אזור 1 — מהימן

אזור 1 — שרת

מה רץ כאן

  • Content API — קטלוג קריאה בלבד + הגשת קבצים למכשירי קצה
  • Admin API — העלאה מאומתת, CRUD מטאדטה, ניהול קטגוריות
  • PostgreSQL — כל מטאדטת הקטלוג (localhost בלבד)
  • מאגר קבצים — סרטונים ו-PDF שהועלו (localhost בלבד)

מאפייני אבטחה

  • סיום HTTPS — nginx על פורט 443; TLS 1.2+
  • DB + מאגר קבצים — localhost בלבד; לא נגישים ישירות לרשת
  • סמכותי — כל שינויי תוכן כאן; הקצה הוא קריאה בלבד
  • הגבלת רשת — הגבל לטווח IP ידוע אם אפשרי (תשתית)

Content API ללא אימות

מכשירי קצה קוראים תוכן באופן אנונימי. ללא middleware אימות ב-/api/catalog או /api/content. ההגבלה היא ברשת, לא בזהות.

אזור 2 — חצי-מהימן

אזור 2 — מכשיר קצה

מודל קיוסק — ללא זהות

  • גישה אנונימית — ללא התחברות, ללא זהות משתמש, פרופיל מכשיר בלבד
  • קריאה בלבד — ניתן למשוך קטלוג + תוכן, לא לשנות נתוני שרת
  • פרופיל מכשיר — בחירת תחומי עניין שמורה מקומית ב-IndexedDB; לא נשלחת לשרת

בידוד רשת

  • Chrome ↔ פרוקסי — HTTP דרך localhost בלבד (אותו מחשב, ללא חשיפת רשת)
  • פרוקסי ↔ ענן — HTTPS; הפרוקסי מאמת תעודת TLS של הענן
  • ללא גישת ענן ישירה מ-Chrome — הפרוקסי הוא נקודת יציאה היחידה

מה אנו שולטים בקצה

  • מיכל Docker — פרוקסי nginx + חבילת SPA (בבעלות הצוות)
  • מטמון פרוקסי — ב-filesystem של ה-VM הלינוקס, לא נגיש ישירות מ-Windows
  • קוד SPA — כל לוגיקת הדפדפן
לא בשליטת הצוות: מערכת הפעלה Windows, תצורת VM לינוקס, Docker daemon, הגדרת מצב קיוסק, גישה פיזית למכשיר. אחסון דפדפן מקומי אינו מוצפן — ארגז החול הוא גבול ה-MVP.

אזור 3 — מאומת

אזור 3 — לקוח מנהל

מודל אימות

  • סיסמה משותפת — פרטי גישה יחידים לכל צוות המנהלים במטה (MVP)
  • טוקן סשן — מוחזר בכניסה; נשלח בכותרת Authorization בכל קריאת Admin API
  • sessionStorage — הטוקן נמחק אוטומטית עם סגירת הלשונית
  • תפוגת טוקן — בצד שרת; טוקנים שפגו תוקפם נדחים

טווח גישה

  • Admin API בלבד — העלאה, CRUD מטאדטה, ניהול קטגוריות
  • רשת מטה — פורטל המנהל לא מיועד להיות ציבורי
  • ללא RBAC — תפקיד יחיד; הרשאות למשתמש הן פוסט-MVP
הערת המשכיות: מודל הסיסמה המשותפת + תפקיד יחיד הוא פשוט במכוון ל-MVP. נתיב שדרוג פוסט-MVP יוסיף חשבונות למשתמש ו-RBAC באותו Admin API — ללא שינויים מבניים.

אבטחת תעבורה

אבטחת תעבורה

חיבור פרוטוקול הערות
Chrome → פרוקסי קצה HTTP localhost בלבד — אותו מחשב פיזי; ללא חשיפת רשת
פרוקסי קצה → שרת ענן HTTPS TLS 1.2+ הפרוקסי מאמת תעודת TLS של הענן; דוחה תעודות לא תקינות
מחשב מטה → שרת ענן HTTPS TLS 1.2+ TLS סטנדרטי של דפדפן; פורטל מנהל מוגש דרך HTTPS
ענן: nginx פורט 443 סיום TLS nginx מסיים TLS; מעביר ל-Node.js על localhost:3000

תעודת TLS (הדגמה)

  • מועדף: Let's Encrypt (חינם, חידוש אוטומטי, אם יש שם דומיין)
  • חלופה: תעודה חתומה עצמית (מקובלת לסביבת מעבדה/הדגמה)

מדוע HTTP ב-localhost מקובל

  • התעבורה נשארת בתוך אותו מחשב פיזי
  • ממשק רשת לא נחצה — loopback בלבד
  • עקבי עם ארכיטקטורת דפדפן/פרוקסי סטנדרטית

אבטחת תוכן

אבטחת תוכן

אימות העלאה (שרת)

  • סוג MIME + סיומת קובץ — שניהם מאומתים בהעלאה; אי-התאמה נדחית
  • ללא קבצים ניתנים להרצה — רק MP4 וידאו ו-PDF סטנדרטי מתקבלים
  • הגבלת גודל — קבצים מעל 100 MB נדחים
  • שמות קבצים שנוצרו — נשמרים כ-{contentId}.{ext}; ללא נתיבים שסופקו על-ידי המשתמש

הגשת קבצים

  • ללא URLs ישירות של filesystem — קבצים מוגשים רק דרך נקודת קצה API
  • API שולט בכותרות — Content-Type, Accept-Ranges, Cache-Control נקבעים על-ידי השרת
  • ללא סכנת path traversal — content-id מאומת לפני חיפוש קובץ

בידוד מטמון קצה

  • מטמון פרוקסי ב-filesystem של ה-VM הלינוקס — לא נגיש מ-Windows
  • Chrome ניגש לקבצים שמטומנים רק דרך HTTP של הפרוקסי — לא ישירות
  • IndexedDB — מאחסן רשומות מטאדטה בלבד; ללא נתוני קבצים גולמיים

ללא טלמטריה

  • אפס נתוני שימוש יוצאים ממכשיר הקצה ב-MVP
  • פעולות לייק/שמירה שמורות מקומית ב-IndexedDB בלבד — ללא סנכרון שרת
  • ללא פלטפורמת אנליטיקה משולבת

היקף אבטחה

היקף MVP — מה לא מכוסה

אחריות תשתית

  • הגדרת קיוסק ברמת מערכת ההפעלה — מצב קיוסק Chrome ונעילת Windows מוגדרים מראש על-ידי צוות הפרישה; לא תוצר הצוות
  • כללי חומת אש VM — קבוצות אבטחת רשת / כללי iptables של ה-VM הם אחריות צוות התשתית
  • אבטחה פיזית — אבטחת הטאבלט מפני גישה פיזית היא אחריות מבצעית

היקף פוסט-MVP

  • DRM / סיווג תוכן — ללא ניהול זכויות דיגיטליות; הגנת התוכן מסתמכת על בידוד רשת + TLS בלבד
  • הצפנת אחסון דפדפן מקומי — IndexedDB ומטמון פרוקסי אינם מוצפנים; ארגז החול הוא גבול ה-MVP
  • אימות משתמש בקצה — קיוסק השדה הוא אנונימי במכוון ב-MVP
  • רישום ביקורת — ללא מסלול ביקורת גישה מעבר ללוגי בקשות שרת סטנדרטיים
לסוקרים: זהו אב-טיפוס מעבדה לסביבת הדגמה. מודל האבטחה מתאים להקשר זה. פרישת ייצור תדרוש הקשחה בתחומים המפורטים לעיל.

סביבת הדגמה

יציבת סביבת ההדגמה

הגדרת מעבדה/הדגמה

  • שרת ענן — נגיש ברשת ידועה; HTTPS עם תעודה חתומה עצמית או Let's Encrypt
  • מכשירי קצה (×N) — כל אחד מריץ מיכל Docker עם פרוקסי nginx + חבילת SPA; Chrome מתחבר ל-localhost:8080
  • פורטל מנהל — נגיש לסוקרים ברשת המטה; לא ציבורי
  • תוכן הדגמה — 10 PDF + 5 סרטונים טעונים מראש; קטלוג מסונכרן לקצה בהפעלה ראשונה

מה המבקרים יכולים לאמת ידנית

  • סמל HTTPS נעול בדפדפן על שרת הענן
  • שורת הכתובת של Chrome מציגה localhost — ללא URL ענן ישיר
  • פורטל מנהל דורש התחברות לפני כל שינוי תוכן
  • העלאת קובץ שאינו MP4/PDF נדחית
  • ניתוק מהרשת — תוכן שנגשו אליו קודם ממשיך לנגן
  • DevTools דפדפן: ללא בקשות רשת חיצוניות ממכשיר הקצה (ללא טלמטריה)
